# Contributing to NEO-Tutorial

Thank you for your interest in contributing to NEO-Tutorial! This document provides guidelines for contributing.

## How to Contribute

### Reporting Issues

- Check existing issues before creating a new one
- Use clear, descriptive titles
- Include steps to reproduce (if applicable)
- Specify which tutorial section is affected

### Submitting Changes

1. Fork the repository
2. Create a feature branch (`git checkout -b feature/your-feature`)
3. Make your changes
4. Commit with clear messages (`git commit -m "Add: description"`)
5. Push to your fork (`git push origin feature/your-feature`)
6. Open a Pull Request

## Content Guidelines

### Writing Style

- Use clear, concise language
- Include code examples where helpful
- Add images/diagrams for complex concepts
- Keep paragraphs short and focused

### Code Blocks

Use proper language identifiers:

```csharp
// C# code
public class Example { }
```

```python
# Python code
def example():
    pass
```

```json
{
  "key": "value"
}
```

### File Structure

- Chinese content: `cn/` directory
- English content: `en/` directory
- Images: Place in `images/` subfolder of relevant section

## Translation

We welcome translations! When translating:

- Maintain the same file structure
- Keep code examples unchanged
- Translate comments in code if helpful
- Preserve all links and references

## Questions?

- Open an issue for discussion
- Join NEO community channels
